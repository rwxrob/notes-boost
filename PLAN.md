# Job-Focused Learning Plan

A roadmap built around getting hired as a junior developer.
Employers want evidence — every stage ends with something you can show.

---

## Stage 1: Terminal, Git & GitHub *(you've started this)*

**Why:** Required for literally every tech job. Shows up in every interview.

- [ ] Navigate the filesystem (`cd`, `ls`, `pwd`, `mkdir`, `rm`, `cp`, `mv`)
- [ ] Read and write files (`cat`, `less`, `nano` or `vim`)
- [ ] Understand pipes and redirection (`|`, `>`, `>>`)
- [ ] Git: `status`, `add`, `commit`, `push`, `pull`, `log`, `diff`
- [ ] Understand branches and pull requests
- [ ] Keep this repo active with regular commits (employers look at GitHub history)

**Portfolio output:** Active GitHub profile with real commit history

---

## Stage 2: Python or Go

**Why:** Python is the most in-demand entry-level language. Go is great for systems/cloud jobs.

- [ ] Variables, types, functions, loops, conditionals
- [ ] Read/write files
- [ ] Handle errors properly
- [ ] Work with JSON and APIs
- [ ] Write at least 3 small projects (not tutorials — your own ideas)

**Portfolio output:** 3+ projects in GitHub repos with READMEs

---

## Stage 3: SQL & Data Basics

**Why:** Almost every job touches a database. SQL shows up in 80%+ of job postings.

- [ ] SELECT, WHERE, JOIN, GROUP BY
- [ ] INSERT, UPDATE, DELETE
- [ ] Basic database design (tables, keys, relationships)
- [ ] Use SQLite or PostgreSQL in a real project

**Portfolio output:** A project that uses a real database

---

## Stage 4: The Web

**Why:** Most software is web-connected. Even backend/devops jobs expect this.

- [ ] How HTTP works (request, response, status codes, headers)
- [ ] REST APIs — call one with `curl` and in code
- [ ] HTML & CSS basics (enough to build a simple page)
- [ ] Build and deploy one web project

**Portfolio output:** A live deployed project with a URL you can share

---

## Stage 5: Cloud & DevOps Basics

**Why:** "Knows how to deploy" separates juniors who get hired from those who don't.

- [ ] Docker basics — what containers are, run one yourself
- [ ] Pick one cloud provider (AWS free tier is a good start)
- [ ] Deploy something using GitHub Actions (CI/CD)
- [ ] Understand environment variables and secrets management

**Portfolio output:** A project that deploys automatically on push

---

## Job Search Prep (do this throughout, not at the end)

- [ ] Write a strong GitHub bio and pin your best repos
- [ ] Build a simple portfolio site (even one page)
- [ ] Practice explaining your projects out loud
- [ ] Do 2-3 mock technical interviews (LeetCode easy, system design basics)
- [ ] Contribute to one open source project (even docs)
